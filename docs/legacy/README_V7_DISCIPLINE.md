# Study Commander V7 — غرفة العمليات وتقارير الانضباط

هذه النسخة تضيف نظام مراقبة تنفيذي لا يعتمد على المزاج، مع تقارير HTML يومية وأسبوعية وإصلاح اتصال Gemini.

## الميزات الجديدة

### 🪖 غرفة العمليات

من القائمة الرئيسية اضغط **🪖 غرفة العمليات**، ثم تستطيع:

- تسجيل تقرير اليوم.
- استلام تقرير HTML احترافي لليوم.
- استلام تقرير HTML لآخر 7 أيام.
- مشاهدة أوامر التصحيح حسب نقاط الضعف.
- تشغيل جلسة إنقاذ فورية 20/5 عند التسويف.

### بيانات التقرير اليومي

يسجل البوت:

- وقت النوم والاستيقاظ ومدة النوم.
- هل الهاتف كان خارج مكان الدراسة.
- دقائق النظري.
- دقائق العملي والصور.
- عدد MCQ وعدد الإجابات الصحيحة.
- عدد الأسئلة المقالية.
- هل تمت مراجعة إغلاق اليوم.
- ملاحظة الطالب.

ثم يحسب `Discipline Score` من 100:

| العنصر | النقاط |
|---|---:|
| النوم | 10 |
| حجر الهاتف | 10 |
| النظري | 20 |
| العملي | 15 |
| MCQ | 20 |
| المقالي | 15 |
| مراجعة الإغلاق | 10 |

التصنيف:

- 85–100: أخضر.
- 70–84: أصفر.
- أقل من 70: أحمر.

### قاعدة البيانات

أضيف جدول جديد:

```text
daily_discipline_reports
```

يُنشأ تلقائيًا عند تشغيل النسخة الجديدة، ولا يحتاج حذف قاعدة البيانات القديمة.

## إصلاح Gemini

تم تحويل الاتصال إلى Gemini Interactions API بدل الاعتماد على المسار القديم فقط.

أضف داخل Railway Variables:

```env
GEMINI_API_KEY=مفتاحك_من_Google_AI_Studio
GEMINI_MODEL=gemini-3.5-flash
GEMINI_API_URL=https://generativelanguage.googleapis.com/v1beta/interactions
GEMINI_FALLBACK_MODELS=gemini-3.1-flash-lite,gemini-3-flash-preview,gemini-2.5-flash
AI_CHAT_ENABLED=true
```

داخل شاشة الذكاء الاصطناعي يوجد زر:

```text
🧪 فحص Gemini
```

يختبر الاتصال ويعرض سبب الفشل بوضوح.

## التحديث على Railway

1. احتفظ بنسخة احتياطية من قاعدة البيانات.
2. استبدل ملفات المستودع بمحتويات هذه النسخة.
3. لا تحذف Railway Variables القديمة.
4. أضف متغيرات Gemini المذكورة أعلاه.
5. ارفع التغييرات إلى GitHub.
6. Railway يعيد النشر تلقائيًا.
7. بعد التشغيل أرسل `/start` وافتح **🪖 غرفة العمليات**.

## ملاحظة أمنية

Gemini لا يحسب الدرجات ولا يعدل التقارير. الحسابات تنفذ بقواعد برمجية ثابتة. Gemini مخصص للشرح، MCQ، المقالي، والمساعدة الدراسية فقط.
